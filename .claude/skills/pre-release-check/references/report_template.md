## 问题条目模板

每个问题条目应遵循以下格式：

```markdown
#### [CODE-001] 函数资源未正确释放

**优先级**: 🔴 关键
**维度**: 代码质量 > 资源管理
**位置**: `src/mediafactory/engine/recognition.py:145-160`
**影响范围**: 可能导致内存泄漏，长时间运行会耗尽系统资源

**问题描述**:
`RecognitionEngine.transcribe()` 方法中加载的 Whisper 模型在异常情况下不会被释放。

当转录过程中抛出异常时（如音频文件损坏），`finally` 块中的清理代码不会执行，导致模型资源泄漏。

**修复建议**:
使用 try-finally 确保资源在任何情况下都会被释放：

```python
# 修复前
def transcribe(self, audio_path: str) -> List[Segment]:
    model = self._load_model()
    result = model.transcribe(audio_path)
    self._release_model(model)
    return result

# 修复后
def transcribe(self, audio_path: str) -> List[Segment]:
    model = self._load_model()
    try:
        return model.transcribe(audio_path)
    finally:
        self._release_model(model)
```

**验证方法**:
1. 使用损坏的音频文件测试转录功能
2. 监控内存使用情况，确认模型被正确释放
3. 添加单元测试验证异常场景

---

**相关代码**:
- `src/mediafactory/engine/recognition.py:145-160`
- `src/mediafactory/pipeline/stages.py:ModelCleanupStage`
```

## 优先级判定标准

### 🔴 关键 (必须修复)

满足以下任一条件：

- **安全漏洞**: 可被利用的安全漏洞（SQL注入、XSS、命令注入等）
- **资源泄漏**: 内存泄漏、文件句柄泄漏、连接泄漏
- **数据损坏**: 可能导致用户数据丢失或损坏
- **系统崩溃**: 可能导致应用程序崩溃
- **功能完全失效**: 核心功能完全无法使用

### 🟠 高优先级 (强烈建议修复)

满足以下任一条件：

- **性能严重退化**: 性能下降超过 50%
- **功能异常**: 核心功能在特定场景下异常
- **架构严重违反**: 严重违反架构原则（如跨层调用）
- **并发问题**: 竞态条件、死锁风险
- **安全风险**: 潜在的安全风险（非直接漏洞）

### 🟡 中优先级 (建议修复)

满足以下任一条件：

- **代码坏味道**: 过长函数、过深嵌套、重复代码
- **可维护性问题**: 代码难以理解和维护
- **测试覆盖不足**: 关键路径缺少测试
- **文档过时**: 文档与代码不一致

### 🟢 低优先级 (可选优化)

满足以下任一条件：

- **命名不一致**: 变量/函数命名不符合约定
- **注释不足**: 缺少必要的注释说明
- **轻微优化**: 小幅性能优化空间
- **代码风格**: 格式、空行等风格问题
